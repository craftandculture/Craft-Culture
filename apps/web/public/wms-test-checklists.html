Flow 1: Receiving

Three-phase receiving: Verify Items → Print Labels → Shelve Stock

1.1 View Pending Shipments

# Action Expected Result Pass
1 Navigate to WMS → Receive See list of pending shipments
2 Verify shipment card shows: transport icon (plane/ship/truck), shipment #, status badge, partner, origin → destination All info displayed correctly with correct transport mode icon
3 Verify status badge colors: Green for "At Warehouse", Amber for "Arrived Port"/"In Clearance" Correct colors displayed
4 Tap on a shipment Opens shipment detail view with item list

1.2 Phase 1: Verify Items

# Action Expected Result Pass
1 View shipment items list See all items with product names, expected quantities
2 Tap on first item Opens item detail view, scrolls to top
3 Verify item info shows: product name, LWIN/SKU, expected cases, pack size Info matches shipment data
4 Verify keyboard does NOT pop up automatically on mobile No keyboard flash on focus
5 Enter received quantity (match expected or adjust) Quantity accepted
6 Tap "Confirm Quantity" Item marked as verified

1.3 Phase 2: Print Labels

# Action Expected Result Pass
1 Tap "Print Labels" button Labels generated, ZPL file downloads
2 Send ZPL to Zebra printer Labels print correctly
3 Verify label shows: QR barcode, product name, pack size, vintage, owner All fields correct and scannable
4 Apply labels to physical cases Labels attached to correct cases

1.4 Phase 3: Shelve Stock

# Action Expected Result Pass
1 See "Scan Bay Barcode" input Input focused, ready for scan
2 Scan location QR code (e.g., LOC-A-01-01) Location recognized, shown on screen
3 Verify location badge shows correct code Badge displays "A-01-01"
4 Tap "Confirm Location" Stock assigned to location, success beep/vibration
5 See success message "Stock assigned to A-01-01" or similar

1.5 Complete Receiving

# Action Expected Result Pass
1 Tap "Next Product" Goes to next incomplete item
2 Repeat phases for remaining items All items received and shelved
3 After last item, tap "Complete Receiving" Shipment marked as received
4 Verify shipment no longer in pending list Shipment removed from list
5 Check Stock page - new stock visible Stock records created with correct locations

1.6 Edge Cases - Receiving

# Action Expected Result Pass
1 Try scanning invalid barcode Error: "Location not found", error buzz/vibration
2 Rapidly scan barcode twice (double-scan) Only processes once, no duplicate
3 Enter 0 cases received Handled gracefully (skip or warn)
4 Close browser mid-receiving, reopen Draft saved, can resume

Flow 2: Stock Check / Cycle Count

2.1 Check Stock By Bay

# Action Expected Result Pass
1 Navigate to WMS → Check (or Ctrl+C) Opens Stock Check page
2 Verify "By Bay" tab is selected by default Tab highlighted
3 See scan input with "Scan bay barcode" label Input visible and ready
4 Scan bay QR code (e.g., LOC-A-01-01) Spinner shows, then results load. Success beep.
5 See location badge with code Shows "A-01-01"
6 See list of stock items at this location Products displayed with quantities
7 Verify each item shows: name, vintage, pack size, LWIN, owner, case count All info correct

2.2 Quick Location Navigation

# Action Expected Result Pass
1 With results showing, look for navigation buttons See prev/next bay, up/down level buttons
2 Tap "Next Bay" button Moves to adjacent bay (A-01-02)
3 Tap "Up Level" button Moves to level above (A-02-01)

2.3 Edit Stock Quantity

# Action Expected Result Pass
1 Tap "Edit Qty" on a stock item Edit mode opens with +/- buttons
2 See current quantity in number input Shows current case count
3 Tap + button Quantity increases by 1
4 Tap - button Quantity decreases by 1
5 See change indicator (e.g., "+2 from 5") Shows delta from original
6 Enter reason (optional) Text accepted
7 Tap "Save" Spinner, then success toast
8 Verify quantity updated on card Shows new quantity

2.4 Reprint Labels

# Action Expected Result Pass
1 See printer icon next to "Edit Qty" button Icon visible
2 Tap printer icon ZPL file downloads
3 Print ZPL on Zebra printer Labels print correctly

2.5 Check Stock By Product

# Action Expected Result Pass
1 Tap "By Product" tab Tab switches
2 See scan input with "Scan case barcode" label Input ready
3 Scan a case label barcode Spinner, then results
4 See product header with name, LWIN, pack size Info correct
5 See "TOTAL STOCK" card with total cases Sum across all locations
6 See "By Location" breakdown Each location with case count

2.6 Edge Cases - Stock Check

# Action Expected Result Pass
1 Scan empty bay (no stock) Shows "No Stock - This location is empty" with "Transfer Stock Here" button
2 Scan invalid location barcode Error: "Location not found: XXX", error buzz
3 Scan case that doesn't exist in WMS "Product Not Found" message
4 Edit quantity to 0 Allowed (stock depleted)
5 Tap "Select from List" Location list appears with search

Flow 3: Picking

Pick items are sorted by location for optimal pick path with "next bay" hints.

3.1 Release Order to Pick

# Action Expected Result Pass
1 Navigate to Admin → Zoho Sales Orders See order list
2 Filter by "Ready for Release" Only invoiced orders shown
3 Click "Release to Pick" on an order Pick list created
4 Navigate to WMS → Pick (or Ctrl+P) See new pick list

3.2 Start Picking

# Action Expected Result Pass
1 Tap on pick list Opens pick list detail
2 Verify items sorted by location for optimal path Items grouped by location, sorted A→Z
3 Tap "Start Picking" or first item Opens picking interface
4 See "GO TO BAY [LOCATION]" instruction Location clearly visible
5 See "Next bay" hint showing upcoming location Helps plan pick path

3.3 Pick Items

# Action Expected Result Pass
1 Go to suggested bay location Physical movement
2 Scan bay QR code to confirm location "Location confirmed", success beep/vibration
3 See "Scan Case Barcode" input Ready for case scan
4 Scan case label barcode Case validated, success feedback
5 If wrong product, see error "Wrong product" warning, error buzz
6 Repeat for quantity needed All cases picked
7 Auto-advances to next item Next product shown with its location

3.4 Complete Pick List

# Action Expected Result Pass
1 After all items picked, see completion screen "All items picked"
2 Tap "Complete Pick List" Pick list completed
3 Stock quantities reduced Check Stock page
4 Order status updated Shows "picked" or "ready to dispatch"

3.5 Edge Cases - Picking

# Action Expected Result Pass
1 Scan wrong location Error: location doesn't match, error buzz
2 Scan case from wrong product Error: product mismatch
3 Scan already-picked case Error or warning
4 Cancel mid-pick Can resume later

Flow 4: Transfer Stock

4.1 Initiate Transfer

# Action Expected Result Pass
1 Navigate to WMS → Transfer (or Ctrl+T) Transfer page opens
2 See "Scan Source Location" input Ready for scan
3 Scan source bay QR code Location loaded, stock shown, success beep
4 Select stock item to transfer Item selected
5 Enter quantity to transfer Quantity accepted

4.2 Complete Transfer

# Action Expected Result Pass
1 See "Scan Destination Location" input Ready for scan
2 Scan destination bay QR code Location validated, success beep
3 Confirm transfer Success message, haptic feedback
4 Check source location Quantity reduced
5 Check destination location Quantity increased
6 Movement recorded in history Movement log updated

Flow 5: Repack

Convert cases between pack sizes (e.g., 12-pack → 6-pack).

5.1 Select Source Stock

# Action Expected Result Pass
1 Navigate to WMS → Repack Repack page opens with step indicator
2 See "Scan Source Location" input Ready for scan
3 Scan bay QR code containing stock to repack Location loaded, stock items shown
4 Select stock item to repack Shows current pack size (e.g., 12x750ml)

5.2 Configure Repack

# Action Expected Result Pass
1 See available target pack sizes Options like 6-pack, 3-pack shown
2 Select target pack size (e.g., 6-pack) Shows conversion: "1 case of 12 → 2 cases of 6"
3 Enter number of source cases to repack Calculates resulting cases
4 Tap "Start Repack" Proceeds to physical repack step

5.3 Physical Repack & Labels

# Action Expected Result Pass
1 See instructions for physical repack "Remove X bottles from case, place in new cases"
2 Physically repack the bottles Bottles moved to new cases
3 Tap "Print New Labels" Labels for new cases download
4 Print and apply new labels New cases labeled with new barcodes

5.4 Assign Destination

# Action Expected Result Pass
1 See "Scan Destination Location" (optional) Can shelve new cases immediately or skip
2 Scan destination bay or skip New cases assigned to location
3 Tap "Complete Repack" Repack completed, success message
4 Verify stock updated Source reduced, new stock created with new pack size
5 Check movements log Shows "repack_out" and "repack_in" movements

Flow 6: Dispatch

Batch picked orders for dispatch to carriers.

6.1 View Dispatch Queue

# Action Expected Result Pass
1 Navigate to WMS → Dispatch See list of picked orders ready for dispatch
2 Verify orders show: order number, customer, case count, pick date All info displayed

6.2 Create Dispatch Batch

# Action Expected Result Pass
1 Tap "New Dispatch Batch" Opens batch creation
2 Select orders to include in batch Orders added to batch
3 Enter carrier/driver info (optional) Info saved
4 Tap "Create Batch" Dispatch batch created

6.3 Complete Dispatch

# Action Expected Result Pass
1 View dispatch batch details See all orders in batch
2 Physically load orders onto vehicle Cases loaded
3 Tap "Mark Dispatched" Batch marked as dispatched
4 Verify order statuses updated Orders show "dispatched"

Flow 7: Zoho Sync

7.1 Sync Stock to Zoho

# Action Expected Result Pass
1 Navigate to WMS → Stock (or Ctrl+S) Stock overview page
2 Click "Sync Zoho" button Modal/confirmation appears
3 Run with "Dry Run" first See what would change
4 Review proposed changes Adjustments listed
5 Run actual sync Inventory adjustment created
6 Check Zoho Inventory Quantities match WMS

Mobile UX Checks

General Mobile Experience

# Action Expected Result Pass
1 All pages responsive on TC27 screen No horizontal scroll needed
2 Touch targets large enough (min 44px) Easy to tap buttons
3 Text readable without zooming Font size appropriate
4 Virtual keyboard hidden by default on scan inputs No keyboard flash on focus
5 Tap keyboard toggle icon to show keyboard when needed Keyboard appears for manual entry
6 Page scrolls to relevant section after actions User sees result

Scanner Integration

# Action Expected Result Pass
1 Scanner input goes to focused field Barcode captured
2 Scanner "Enter" key triggers submit No manual tap needed
3 Rapid scanning handled (debounce) No double-processing
4 Manual entry possible when needed Can type barcodes after tapping keyboard icon

Sound & Haptic Feedback

# Action Expected Result Pass
1 Successful scan Success beep sound + single vibration (100ms)
2 Error (invalid barcode, wrong product) Error buzz sound + double vibration pattern (100-50-100ms)
3 Device volume up Sounds audible in warehouse environment

Dashboard Features

# Action Expected Result Pass
1 Tap dark mode toggle (sun/moon icon) Theme switches between light and dark
2 Drag quick action tiles to reorder Order saved, persists on refresh
3 Go offline (airplane mode) Offline banner appears at top
4 Come back online Online banner appears briefly, then disappears
