System Overview

The WMS is a custom warehouse management system built specifically for Craft & Culture to manage wine inventory with barcode scanning and multi-owner stock tracking.

Key Concepts

Concept Description
Location A bin in the warehouse (e.g., A-01-02 = Aisle A, Bay 01, Level 02)
LWIN-18 Unique product code including vintage and pack config
Case Label Barcode sticker on each case for tracking
Movement Any stock action (receive, transfer, pick, etc.) - the audit trail
Owner Partner who owns the stock (consignment or purchased)

Workflows

1. Receiving a Shipment

When goods arrive at the warehouse:

1
Navigate to Receive
Go to WMS → Receive and select the pending shipment
2
Verify Quantities
Enter the actual received quantity for each item. Mark any discrepancies.
3
Assign Locations
For each product, scan or select the bay where it will be stored.
4
Complete Receiving
Click complete. Stock records and movement history are created.

2. Stock Reconciliation

If stock numbers don't match what movements say should be there:

1
Check Reconciliation
Go to Stock → Reconcile. See expected vs actual stock.
2
Fix Issues
  • Stock Over (too many): Click "Auto-Fix" to remove orphan/duplicate records
  • Stock Under (too few): Click "Rebuild Stock" to recreate from movements

3. Transferring Stock

Moving cases between locations:

1
Go to Transfer
WMS → Transfer
2
Scan Source
Scan a case barcode or location barcode
3
Scan Destination
Scan the new location barcode
4
Confirm
Movement is logged automatically

Testing Checklist

Before Using in Production

  • Dashboard shows correct totals (Total Cases, Products, Locations)
  • Stock by Owner shows partner breakdown
  • Receive a test shipment end-to-end
  • Verify stock appears at correct location after receiving
  • Transfer stock between two locations
  • Verify movement history records both actions
  • Run reconciliation - should show "Stock is Reconciled"

Hardware Testing (When TC27 + ZD421 Ready)

  • TC27 scanner connects to web app in Chrome
  • Scan location barcode → shows in UI
  • Scan case barcode → shows product details
  • Print case label → ZD421 prints correctly
  • Printed barcode scans correctly
  • Put-away workflow works with scanner
  • Transfer workflow works with scanner

Known Issues to Watch

  • Pack variants: If received pack (6x750ml) differs from expected (12x750ml), verify LWIN-18 is correct
  • Duplicate stock: Should be impossible now due to database constraints - if it happens, report immediately
  • Dashboard zeros: If dashboard shows 0 but movements exist, use "Rebuild Stock" on reconciliation page

Hardware Setup

Zebra TC27 Scanner

  1. Install Chrome browser
  2. Install "Zebra Browser Print" app from Play Store
  3. Configure scanner for "Keyboard Wedge" mode (settings may vary)
  4. Navigate to cc.craftandculture.ae and log in

Zebra ZD421 Label Printer

  1. Pair with TC27 via Bluetooth
  2. Load 4" x 2" direct thermal labels
  3. Run calibration (auto-calibrate for label size)
  4. Open Zebra Browser Print app on TC27 and connect to printer

Label Formats

Label Type Size Content
Case Label 4" x 2" Barcode, Product Name, LWIN, Pack Size, Bay
Location Label 4" x 2" Barcode, Location Code (A-01-02)

Troubleshooting

Dashboard shows all zeros

Cause: Stock records missing or deleted
Fix: Go to Stock → Reconcile → Click "Rebuild Stock"

Reconciliation shows discrepancy

If actual > expected (too many cases):
Click "Auto-Fix" to remove orphan/duplicate records

If actual < expected (too few cases):
Click "Rebuild Stock" to recreate from movements

Labels not printing

Check:
  • Zebra Browser Print app is running on TC27
  • Printer is paired via Bluetooth
  • Printer has labels loaded and is online
  • Try printing a test page from Zebra Browser Print app first

Scanner not working

Check:
  • Scanner is in "Keyboard Wedge" mode
  • Cursor is in the scan input field before scanning
  • Barcode is not damaged or obscured
