Section 7

What Changes, What Stays

The nav restructure is a wrapper change. Existing page components don't change at all.

Remove
❌ Admin sidebar (14 items in a permanent column)
❌ Header pill groups with pipe separators
❌ Conditional nav rendering in layout.tsx (200+ lines)
❌ AdminNav.tsx (legacy horizontal bar)
❌ Duplicate navigation paths (same page reachable 3 ways)
Add
✓ Top nav with 5-6 role-aware items
✓ Tab bar per section (replaces sidebar + pills)
✓ Home dashboard with agent briefing feed
✓ Mobile drawer with same IA
✓ Command bar (⌘K) for power-user navigation
Keep Unchanged
→ All existing page components (tables, forms, dashboards)
→ WMS card grid dashboard (works well)
→ WMS scanner mode layout (separate concern)
→ All tRPC routes and API endpoints
→ Design tokens, Card, Typography, Badge, Button
Effort Estimate
New components: ~4 (TopNav, TabBar, MobileDrawer, HomeDashboard)
Modified files: ~3 (platform layout, mobile nav, footer)
Deleted files: ~2 (AdminSidebar, AdminNav)
Net lines: Likely negative (removing more than adding)
Risk: Low — wrapper-only changes, no business logic touched
